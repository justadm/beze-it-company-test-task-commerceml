# Тестовое задание от BEZE IT Company

Экспорт раздела каталога Bitrix вместе со вложенными разделами, товарами и SKU в CommerceML-пакетах с отправкой по штатному протоколу 1С через `/bitrix/admin/1c_exchange.php`.

## Структура

- `bootstrap.php` — подключение ядра и автозагрузки.
- `local/export/commerceml/send.php` — CLI-точка входа для CommerceML-обмена.
- `src/CommerceML/` — генерация CommerceML и отправка в `1c_exchange.php`.
- `docs/architecture.md` — схема решения.
- `docs/run.md` — пример запуска.

## План

- определить, является ли инфоблок каталогом;
- определить, есть ли у товаров SKU;
- сформировать CommerceML `import_XXX.xml` по выбранному разделу и его поддереву;
- при наличии SKU сформировать `offers_XXX.xml` для тех же родительских товаров;
- разбить товары на пакеты до 2000 родительских товаров;
- пройти штатный протокол `checkauth -> init -> file -> import -> complete` через `Bitrix\Main\Web\HttpClient`.
- поддержать `dryRun` для локальной генерации CommerceML без отправки.

## Ограничения задания

- только методы ядра, без REST API;
- POST-запрос без `curl`;
- код принимающей стороны не реализуется;
- пакет должен быть самодостаточным.

## Принимающая сторона

- Принимающая сторона предполагается полностью штатной.
- Используется стандартный endpoint `/bitrix/admin/1c_exchange.php?type=catalog`.
- На нашей стороне реализуется только генерация CommerceML и клиент пошагового обмена.
