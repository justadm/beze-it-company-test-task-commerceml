# Постановка задачи

Нужно реализовать перенос раздела каталога со всеми вложенными разделами и товарами с одного Битрикса в другой с сохранением структуры каталога.

## Ограничения

1. Использовать только методы ядра. Без REST API.
2. Максимальное количество товаров в передаваемом пакете — 2000.
3. Передача данных должна выполняться POST-запросом. `curl` не использовать.
4. Передавать основные данные разделов и товаров.
5. Код принимающей стороны писать не требуется.
6. Передаваемый пакет должен быть самодостаточным для восстановления структуры и связей.

## Принятая интерпретация

В качестве штатного механизма приема на стороне Bitrix выбран стандартный сценарий CommerceML через `/bitrix/admin/1c_exchange.php?type=catalog`.

Поэтому в рамках реализации:

- на стороне отправителя формируются CommerceML-файлы;
- на стороне приемника используется стандартный импорт Bitrix;
- логика решения строится вокруг штатного протокола `checkauth -> init -> file -> import -> complete`.

## Дополнительные допущения

- ограничение `2000` применяется к родительским товарам;
- если у каталога настроен инфоблок торговых предложений, предложения отправляются вместе с соответствующим пакетом родительских товаров;
- если реальные торговые предложения отсутствуют, `offers_XXX.xml` не формируется;
- CLI выбран как технический способ запуска, а не как архитектурное ограничение решения.
